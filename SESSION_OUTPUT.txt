COMMIT 3: 98a0868 — feat(safety): benchmark freshness governance
================================================================================

Problem: All benchmark data hardcoded with dataDate: '2024-12-01' (~16 months
stale). Evaluator generated "above market average" conclusions without
checking data age.

Solution: 3-state freshness system preventing definitive conclusions from
outdated data.

States:
  current  (≤180 days)  — full comparison, normal language
  aging    (181-365 days) — comparison shown with data date amber warning
  stale    (>365 days)  — confidence downgraded one step, definitive language
                          replaced with hedged/historical wording

Type changes (src/lib/policy-evaluation/types.ts):
  - Added BenchmarkFreshness type: 'current' | 'aging' | 'stale'
  - Extended BenchmarkConfidence with freshness?, dataAsOf?, dataAgeDays?
  - Added benchmarkAgingDays?, benchmarkStaleDays? to EvaluationConfig
    (defaults: 180, 365)

Config changes (src/lib/config/types.ts):
  - Added benchmarkAgingDays: 180, benchmarkStaleDays: 365 to
    EvaluationConfig and DEFAULT_EVALUATION_CONFIG

Core logic (src/lib/policy-evaluation/evaluator.ts):
  - Added computeBenchmarkFreshness(dataDate, agingDays, staleDays)
  - Modified assessBenchmarkConfidence() — now accepts dataDate and
    evalConfig, computes freshness, downgrades confidence when stale
    (high→low, low→suppressed)
  - Modified evaluatePolicy() — gets dataDate from benchmark, passes to
    assessBenchmarkConfidence()
  - Modified evaluatePremium() — gates issue strings by freshness:
    Stale: "above market estimate" → "above historical market estimate
           (data from 2024-12-01) — updated validation recommended"
    Stale: "exceeds typical market range" → "exceeds historical market
           range (data from 2024-12-01) — updated validation recommended"
    Current/aging: normal language unchanged
  - Modified disclaimer generation:
    Stale: "Benchmark data is from {date} ({N} days old). This is a
           historical reference only — updated market validation recommended."
    Aging: "Market data from {date}. Actual premiums vary by vehicle,
           driver profile, and region."
    Current: "Market averages are estimates based on indicative data.
             Actual premiums vary by vehicle, driver profile, and region."

Benchmark service (src/lib/policy-evaluation/benchmark-service.ts):
  - Added dataDate?: string to LegacyPremiumRange interface
  - Populated dataDate in DB path (${match.year}-01-01) and hardcoded
    fallback path ('2024-12-01')

UI (src/components/PolicyDetailView.tsx):
  - Freshness badge in Market Comparison card (desktop + mobile):
    Stale: Red dot + red bg, "Benchmark data is {N} days old (from {date}).
           Historical reference only."

# 10. Translation Hook Implicit Typings\n\nDate: 2026-03-07\n\n## Context\nDuring the FX UI cosmetic sweep, an arbitrary localized string 'approx.' was injected into a React component relying on `t.common.approx`. Because the base interface lacked the explicit definition, it caused the CI pipeline's strict TypeScript compiler (`tsc --noEmit`) to fail.\n\n## Decision\nTo maintain the 0-error strict code baseline, we mandate that any new translation strings introduced dynamically on the frontend MUST either:\n1. Be explicitly defined in the typescript dictionary interfaces before usage.\n2. Leverage pre-existing, type-safe fallback strings.\n\nAd-hoc implicit typing of the `t` variable from `useTranslation()` will be structurally rejected by the build system to guarantee safety.
